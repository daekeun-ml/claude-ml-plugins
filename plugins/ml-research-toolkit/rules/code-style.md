# Code Style

코드를 쓰거나 고칠 때 항상 적용. (연구 코드: 학습 스크립트·평가·해석성 분석·노트북 포함.)

## 일반
- **주변 코드를 닮게.** 기존 파일의 네이밍·주석 밀도·import 순서·포맷·관용구에 맞춘다. 새 스타일을 들여오지 않는다.
- **작은 diff.** 요청 범위만 건드린다. 무관한 리팩터·포맷 대량 변경 금지.
- **죽은 코드 금지.** 미사용 import/변수/함수, 주석처리된 코드 덩어리 남기지 않는다.
- **주석은 "왜".** 무엇을 하는지가 아니라 왜 그렇게 하는지. 자명한 코드에 장황한 주석 금지.
- **에러를 삼키지 말 것.** 빈 except/무시된 실패 금지. 실패는 드러나게.

## Python / ML (주 사용 스택)
- **타입 힌트** 공개 함수 시그니처에. **f-string** 사용. **pathlib** 선호.
- **PEP8** 기본, 프로젝트에 formatter(black/ruff) 설정 있으면 그걸 따른다.
- **랜덤/재현성**: seed 고정 지점 명시. 실험 코드는 config(하이퍼파라미터)를 상단/별도로.
- **디바이스·dtype 명시**: `.to(device)` 일관, CPU/GPU 분기는 명확히. seed 고정으로 재현성 확보.
- **시크릿·경로**: 하드코딩 금지 → env var / 플레이스홀더. 노트북은 출력에 자격증명 흘리지 않기.
- **대용량 자원**: 파일 핸들·세션·CUDA 메모리·엔드포인트는 확실히 정리(context manager / cleanup).

## 대상별 눈높이 (중요)
- **Jupyter 노트북 = 초심자용.** 쉽고 간결하게. **디폴트 설명 언어는 한국어**(markdown 셀·주석). 한 셀은 한 가지만, 마법 같은 한 줄보다 풀어 쓴 여러 줄. 왜 하는지 markdown으로 먼저, 그다음 코드. 전문 용어는 처음 등장 시 한 줄 풀이. 과한 추상화·헬퍼 남발 금지.
- **스크립트(.py) = 그 정도로 친절할 필요 없음.** 대신 **정확하고 직관적**으로. 명확한 네이밍·간결한 구조·예측 가능한 흐름. 주석은 "왜"에 집중, 자명한 건 생략. 실무자가 읽는다고 가정.

## 노트북
- 셀 순서 논리적으로(설치→설정→데이터→실행→확인→정리). 상태 의존(순서 실행 강제)은 최소화.
- 무거운/과금 셀은 주석으로 경고. 재실행 안전하게(idempotent) 지향.
